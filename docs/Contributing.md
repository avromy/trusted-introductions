# Contributing

## Workflow

1. Read `README.md` and relevant docs.
2. Check `docs/Milestones.md` and `docs/Backlog.md`.
3. Create a focused branch.
4. Update docs alongside implementation.
5. Run available checks.
6. Open a PR using the pull request template.

## Decision Records

Create an ADR in `docs/TechnicalDecisions/` for significant product, architecture, privacy, or operational decisions.

## Handoffs

Use `docs/handoffs/EngineeringHandoffTemplate.md` when pausing work or transferring context to another engineer or Codex session.
